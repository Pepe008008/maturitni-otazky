## 1. K čemu slouží programovací jazyk?
Programovací jazyk slouží jako prostředník mezi člověkem a počítačem. Počítač rozumí pouze strojovému kódu (jedničkám a nulám). Programovací jazyk nám dává sadu srozumitelných pravidel (syntaxi) a slovíček, pomocí kterých dokážeme zapsat logiku a algoritmy. Tento náš zápis (zdrojový kód) se následně přeloží (kompiluje nebo interpretuje) do strojového kódu, který počítač vykoná.

## 2. Rozdíl mezi objektově orientovaným a procedurálním jazykem
- **Procedurální programování:** Program je vnímán jako **seznam instrukcí a funkcí**. Data (proměnné) a logika (funkce), která s nimi pracuje, jsou oddělené. Data často "plavou" programem a předávají se z funkce do funkce. Je to skvělé pro menší skripty, ale u velkých projektů se kód stává nepřehledným.
- **Objektově orientované programování (OOP):** Program je vnímán jako **spolupráce objektů**. Data a logika nejsou oddělené, ale jsou "zabalené" (zapouzdřené) uvnitř objektů. Objekt si nese svůj stav (data) a umí s ním sám pracovat (metody). To usnadňuje údržbu kódu, jeho znovupoužitelnost (DRY) a modelování reálného světa.

## 3. Příklady OOP programovacích jazyků
- **Čistě nebo primárně objektové:** Java, C#, Ruby
- **Multiparadigmatické (podporují OOP i jiné přístupy):** Python, C++, PHP, JavaScript

## 4. Základní pojmy OOP: Třída a Instance

- **Třída (Class):** Je to **předpis, šablona nebo plánek**. Představ si ji jako formičku na cukroví nebo technický výkres auta. Třída sama o sobě nic nedělá, jen definuje, jaké vlastnosti (atributy) a schopnosti (metody) bude mít objekt, který podle ní vytvoříme.
- **Instance třídy (Objekt):** Je to konkrétní, **hmatatelný produkt** vytvořený podle třídy. Pokud je třída formička, instance je to upečené cukroví. Můžeme vytvořit stovky instancí z jedné třídy – každá bude mít stejné chování, ale může mít jiné vnitřní hodnoty (např. jedno auto bude červené, druhé modré).

### Další klíčové pojmy (Pilíře OOP):
- **Atributy:** Proměnné patřící danému objektu (vnitřní stav).
- **Metody:** Funkce patřící danému objektu (vnitřní chování).
- **Zapouzdření (Encapsulation):** Skrytí vnitřních dat objektu před okolním světem, aby je nikdo nemohl omylem rozbít. Komunikace probíhá jen přes povolené metody.
- **Dědičnost (Inheritance):** Schopnost vytvořit novou třídu na základě existující. Nová třída převezme všechny vlastnosti a metody a může si přidat vlastní.

## 5. & 6. Soubor `hrdina.py` - Ukázka OOP konstrukcí

Jelikož nemám tvůj původní soubor, vytvořil jsem ukázkový soubor `hrdina.py`, na kterém přesně popíšu všechny OOP konstrukce, na které ses ptal.

```python
# --- SOUBOR: hrdina.py ---

# 1. TŘÍDA (Základní šablona)
class Postava:
    def __init__(self, jmeno, zdravi):
        # ATRIBUTY (vnitřní stav objektu)
        self.jmeno = jmeno
        self.zdravi = zdravi
    
    # METODA (vnitřní chování objektu)
    def predstav_se(self):
        print(f"Jsem {self.jmeno} a mám {self.zdravi} HP.")

# 2. DĚDIČNOST (Hrdina dědí z Postavy)
class Hrdina(Postava):
    def __init__(self, jmeno, zdravi, sila):
        # Volání inicializace nadřazené třídy
        super().__init__(jmeno, zdravi)
        
        # ZAPOUZDŘENÍ (Dvě podtržítka znamenají "privátní" atribut)
        # K tomuto atributu by se nemělo přistupovat zvenčí přímo!
        self.__zkusenosti = 0 
        self.sila = sila
    
    # METODA
    def utok(self):
        print(f"{self.jmeno} útočí silou {self.sila}!")
        self.__pridej_zkusenosti(10) # Volání privátní metody
        
    # ZAPOUZDŘENÍ (Privátní metoda - slouží jen pro vnitřní potřeby třídy)
    def __pridej_zkusenosti(self, mnozstvi):
        self.__zkusenosti += mnozstvi
        if self.__zkusenosti >= 100:
            print(f"{self.jmeno} postoupil na další úroveň!")

# 3. INSTANCE (Konkrétní objekty vytvořené z třídy)
aragorn = Hrdina("Aragorn", 100, 25)
gimli = Hrdina("Gimli", 120, 20)

# Práce s objekty pomocí jejich metod
aragorn.predstav_se()  # Zděděná metoda z třídy Postava
aragorn.utok()         # Metoda třídy Hrdina

# aragorn.__zkusenosti = 100  # Toto by vyhodilo chybu, atribut je zapouzdřený (privátní)
```

## 8. Magické metody (Dunder methods) v Pythonu

Magické metody (mají na začátku a konci dvě podtržítka, angl. *double underscore* = *dunder*) neslouží k tomu, abychom je volali my přímo (např. `objekt.__init__()`), ale **volají se automaticky** při určitých událostech nebo použití operátorů.

- `__init__(self)`: Konstruktor. Volá se automaticky, když se vytváří nová instance. Slouží k nastavení počátečních hodnot (atributů).
- `__str__(self)`: Volá se automaticky, když chceme objekt vypsat (např. pomocí `print(objekt)`). Definuje, jak má objekt vypadat jako text.
- `__sub__(self, other)`, `__add__(self, other)`: Volají se při použití matematických operátorů (`-`, `+`) mezi objekty. Umožňují nám definovat, co se stane, když sečteme nebo odečteme dvě naše instance (tzv. přetěžování operátorů).

```python
class Vektor:
    def __init__(self, x, y):
        self.x = x
        self.y = y
        
    # Co se stane, když objekt vypíšeme
    def __str__(self):
        return f"Vektor({self.x}, {self.y})"
        
    # Co se stane, když použijeme operátor '-' (odčítání / __sub__)
    def __sub__(self, jiny_vektor):
        nove_x = self.x - jiny_vektor.x
        nove_y = self.y - jiny_vektor.y
        return Vektor(nove_x, nove_y)

v1 = Vektor(5, 5)
v2 = Vektor(2, 3)

# Zde se na pozadí automaticky volá v1.__sub__(v2)
vysledek = v1 - v2 

# Zde se na pozadí automaticky volá vysledek.__str__()
print(vysledek) # Vypíše: Vektor(3, 2)
```

## 7. Událostmi řízené programování (Event-driven programming)

Zatímco klasický program běží lineárně shora dolů a pak skončí, událostmi řízený program funguje jinak. Nastartuje se a pak **čeká** v nekonečné smyčce (Event loop), dokud se něco nestane. 

- **Událost (Event):** Nějaký podnět zvenčí. Kliknutí myší, stisk klávesy, přijetí dat ze sítě, vypršení časovače.
- **Obsluha události (Event Handler / Callback):** Funkce nebo metoda, kterou si programátor připraví, a systém ji *automaticky zavolá* ve chvíli, kdy daná událost nastane.

**Ukázka `udalosti.py` (Základní koncept bez složitých knihoven):**
V tomto teoretickém modelu registrujeme funkci, která čeká na událost.

```python
# --- SOUBOR: udalosti.py ---

# Toto je naše OBSLUHA UDÁLOSTI (Handler)
def pri_kliknuti_na_tlacitko(souradnice_x, souradnice_y):
    print(f"Byl zaznamenán klik na souřadnicích {souradnice_x}, {souradnice_y}.")
    print("Spouštím stahování...")

# Simulace systému, který registruje naši událost (tzv. Binding / naslouchání)
# Systému říkáme: "Když nastane událost 'KLIK_MYSI', zavolej moji funkci 'pri_kliknuti_na_tlacitko'"
system_udalosti = {"KLIK_MYSI": pri_kliknuti_na_tlacitko}

# Simulace Event loopu (hlavní smyčky programu)
print("Program běží a čeká na událost...")
# Zde by v realitě program zamrzl a čekal na uživatele.
# Simulujeme, že uživatel klikl na souřadnice (150, 200)
vyvolana_udalost = "KLIK_MYSI"
if vyvolana_udalost in system_udalosti:
    handler_funkce = system_udalosti[vyvolana_udalost]
    handler_funkce(150, 200) # Systém volá naši obsluhu
```

## 9. Ukázka v Tkinter na OOP (Spojení OOP a událostí)

Knihovna `tkinter` pro grafická rozhraní (GUI) v Pythonu je dokonalým příkladem, jak se spojuje OOP a událostmi řízené programování. Celá aplikace je objekt (třída) a reaguje na události (kliky na tlačítka).

```python
import tkinter as tk

# Aplikace je definována jako TŘÍDA, která udržuje svůj vnitřní stav (atribut kliknuti)
class MojeAplikace:
    def __init__(self, koren):
        self.koren = koren
        self.koren.title("Ukázka OOP a Událostí")
        
        # ATRIBUT pro uchování dat
        self.pocet_kliknuti = 0
        
        # Vytvoření grafického prvku
        self.popisek = tk.Label(koren, text="Zatím jsi neklikl.")
        self.popisek.pack(pady=10)
        
        # Tlačítko - Zde propojujeme UDÁLOST s OBSLUHOU UDÁLOSTI (handlerem)
        # Událost: Klik na tlačítko. Handler: self.obsluha_kliku (naše metoda)
        self.tlacitko = tk.Button(koren, text="Klikni na mě!", command=self.obsluha_kliku)
        self.tlacitko.pack(pady=10)

    # METODA, která slouží jako OBSLUHA UDÁLOSTI
    def obsluha_kliku(self):
        self.pocet_kliknuti += 1
        # Změna stavu objektu na základě události
        self.popisek.config(text=f"Klikl jsi {self.pocet_kliknuti} krát.")

# Vytvoření základu okna (hlavní smyčka událostí z tkinteru)
root = tk.Tk()
root.geometry("300x150")

# Vytvoření INSTANCE naší aplikace
aplikace = MojeAplikace(root)

# Spuštění EVENT LOOPu (program nyní běží a čeká na události - kliknutí)
root.mainloop() 
```


## 10. Obor platnosti proměnných (Scope): Globální, lokální a privátní

Problematika privátních, lokálních a globálních proměnných úzce souvisí s tím, čemu se v programování říká **obor platnosti (scope)**. Jde o to, ze kterých částí kódu máme k dané proměnné přístup a kde naopak "neexistuje". 

Rozdělíme si to na tři hlavní kategorie:

### 10.1 Globální proměnné (Global variables)
Globální proměnná je definována v hlavním těle programu (mimo jakoukoliv funkci nebo třídu). Můžeš ji číst naprosto **odkudkoliv** – z hlavní části kódu, zevnitř funkcí i zevnitř objektů.

**Pravidlo dobrého kódu:** Globálním proměnným se snažíme vyhýbat, jak to jen jde. Pokud je může změnit kdokoliv a odkudkoliv, v rozsáhlém kódu velmi rychle ztratíš přehled o tom, proč se hodnota změnila, a vznikají těžko odhalitelné chyby (tzv. špagetový kód).

```python
# Toto je GLOBÁLNÍ proměnná
uzivatelske_jmeno = "Admin"

def vypis_uzivatele():
    # Funkce má ke čtení globální proměnné bez problémů přístup
    print(f"Přihlášen: {uzivatelske_jmeno}")

def zmen_uzivatele(nove_jmeno):
    # POZOR: Pokud chceme globální proměnnou uvnitř funkce ZMĚNIT, 
    # musíme Pythonu explicitně říct, že myslíme tu globální.
    global uzivatelske_jmeno
    uzivatelske_jmeno = nove_jmeno

vypis_uzivatele()      # Vypíše: Přihlášen: Admin
zmen_uzivatele("Karel")
vypis_uzivatele()      # Vypíše: Přihlášen: Karel
```

### 10.2 Lokální proměnné (Local variables)
Lokální proměnná vzniká **uvnitř nějaké funkce** (nebo metody) a existuje **pouze tam**. Jakmile funkce skončí, proměnná je z paměti vymazána. Zbytek programu o ní vůbec neví.

Slouží k provádění výpočtů a dočasnému ukládání dat, která nepotřebujeme nikde jinde.

```python
def spocitej_obsah_obdelniku(a, b):
    # 'vysledek', 'a', 'b' jsou LOKÁLNÍ proměnné
    vysledek = a * b
    return vysledek

obsah = spocitej_obsah_obdelniku(5, 10)
print(f"Obsah je {obsah}")

# Pokud bychom se pokusili vypsat proměnnou 'vysledek' tady venku, 
# program spadne na chybu (NameError), protože zde už neexistuje.
# print(vysledek)  <-- TOTO ZPŮSOBÍ CHYBU
```

### 10.3 Privátní proměnné / atributy (Private variables v OOP)
S pojmem "privátní" se setkáváme především v **objektově orientovaném programování (OOP)**. Jak jsme si ukázali u zapouzdření, chceme chránit vnitřní stav (atributy) objektu před zásahem zvenčí.

V jazycích jako Java nebo C# k tomu slouží klíčové slovo `private`. Python to řeší specificky: používá **dvě podtržítka `__` na začátku názvu atributu**. Tím říkáme: *"Tohle je vnitřní záležitost tohoto objektu, nesahej na to zvenčí!"*

Abychom k takovým datům mohli přistupovat (nebo je měnit) bezpečně, vytváříme tzv. **gettery** (metody pro čtení) a **settery** (metody pro zápis), které obsahují kontrolní logiku.

```python
class BankovniUcet:
    def __init__(self, majitel, pocatecni_zustatek):
        self.majitel = majitel               # Veřejný atribut (lze číst i měnit zvenčí)
        self.__zustatek = pocatecni_zustatek # PRIVÁTNÍ atribut (skrytý uvnitř objektu)

    # Getter - metoda pro bezpečné přečtení privátní hodnoty
    def zjisti_zustatek(self):
        return self.__zustatek

    # Setter - metoda pro bezpečnou změnu privátní hodnoty (vklad peněz)
    def vloz_penize(self, castka):
        if castka > 0:
            self.__zustatek += castka
            print(f"Vloženo {castka} Kč.")
        else:
            print("Chyba: Nemůžeš vložit zápornou částku!")

# Vytvoření objektu
muj_ucet = BankovniUcet("Jan Novák", 1000)

print(muj_ucet.majitel)          # Funguje (vypíše: Jan Novák)
# print(muj_ucet.__zustatek)     # CHYBA! (AttributeError) Atribut je privátní.

# Získání a změna hodnoty přes povolené metody (ZAPOUZDŘENÍ)
print(muj_ucet.zjisti_zustatek()) # Funguje (vypíše: 1000)
muj_ucet.vloz_penize(500)         # Funguje bezpečně přes metodu
```

### Shrnutí v kostce

| Typ proměnné | Kde se definuje? | Kde je dostupná? | Kdy ji použít? |
| :--- | :--- | :--- | :--- |
| **Globální** | Volně v hlavním kódu | Úplně všude | Skoro nikdy (nebo jako konstanty, např. `PI = 3.14`). |
| **Lokální** | Uvnitř funkce | Jen v té dané funkci | Vždy, když potřebuješ pomocnou proměnnou pro výpočet uvnitř funkce. |
| **Privátní** | Uvnitř třídy (s `__`) | Jen uvnitř metod dané třídy | V OOP, pro ochranu dat objektu (zapouzdření). |
