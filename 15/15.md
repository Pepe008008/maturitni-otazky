# 15. Internetové technologie

***Obsah otázky:*** Pojmy Internet, WWW, doména, webhosting, cloud. Zápis kódu webových stránek, HTML (struktura, základní značky), CSS (využití při tvorbě webových stránek, nástroje v prohlížečích). Jazyky interpretované na straně prohlížeče, jazyky interpretované na straně serveru.

## Pojmy
- **Internet** - globální počítačová síť skládající se z propojených sítí. Není to jedna centrální síť, ale "síť sítí".
    - **TCP/IP protokol:** Základní kámen internetu. 
        - *IP (Internet Protocol)* se stará o adresování (každé zařízení má svou IP adresu, např. IPv4 `192.168.1.1` nebo modernější IPv6) a směrování cesty.
        - *TCP (Transmission Control Protocol)* data rozseká na malé "balíčky" (pakety), pošle je a na konci zkontroluje, jestli dorazily všechny a ve správném pořadí. Pokud ne, vyžádá si je znovu.
- **WWW (World Wide Web)** - informační systém dostupný přes internet. Zatímco *internet* je fyzická a síťová infrastruktura (kabely, routery), *WWW* je služba, která nad tím běží (webové stránky, odkazy).
- **Doména** - slovní označení IP adresy počítače, aby si uživatelé nemuseli pamatovat čísla.
    - **DNS (Domain Name System):** Funguje jako "telefonní seznam" internetu. Když zadáš `seznam.cz`, tvůj prohlížeč se zeptá DNS serveru: "Jakou IP adresu má seznam.cz?". DNS server odpoví (např. `77.75.79.53`) a prohlížeč tě tam připojí.
    - Českou top-level domain (TLD) `.cz` spravuje sdružení **CZ.NIC**.
    - **Registrátor domén** - společnost (např. Forpsi, Wedos, Active24), která má smlouvu s CZ.NIC a zprostředkovává ti pronájem domény na určité období (obvykle rok).
- **Webhosting** - pronájem prostoru pro webové stránky na cizím serveru. Sdílíš výkon jednoho fyzického serveru s dalšími stovkami webů (je to levné, ale pokud má cizí web obrovskou návštěvnost, může to zpomalit i tvůj).
- **Cloud** - modernější alternativa k webhostingu. Tvůj web neběží na jednom fyzickém stroji, ale na rozsáhlé síti propojených serverů (např. AWS, Google Cloud, Microsoft Azure). 
    - *Výhody:* Obrovská spolehlivost (když spadne jeden server, okamžitě to převezme jiný) a škálovatelnost (platíš jen za tolik výkonu, kolik zrovna tvůj web spotřebovává).




## Webový server a HTTP komunikace
- **Protokol HTTP/HTTPS:** Způsob, jakým se prohlížeč (klient) a server domlouvají. Je to protokol "požadavek-odpověď".
- **HTTP Metody:**
    - `GET`: Klient si žádá o data (např. načtení článku).
    - `POST`: Klient posílá nová data na server (např. odeslání přihlašovacího formuláře).
- **HTTP Status kódy (odpověď serveru):**
    - `200 OK`: Vše proběhlo v pořádku, posílám web.
    - `400 Bad Request`: Chyba na straně klienta (špatný požadavek).
    - `404 Not Found`: Stránka nenalezena (např. překlep v URL).
    - `500 Internal Server Error`: Chyba na straně serveru (např. spadla databáze).


## Webový server a Frameworky
- **Webový server (Software):** Program, který umí přijímat a odpovídat na HTTP požadavky (naslouchají na 80 nebo 443). Jeho hlavním úkolem je posílání webových souborů klientovi přes internet.
- **Nejpoužívanější webový server: Nginx** (nebo Apache)
    - Buď čte soubory přímo z disku a posílá je.
    - Nebo může HTTP požadavek předat jinému serveru na pozadí a jeho odpověď předat zpět (funguje jako **proxy**).
    - **Load balancing:** Můžeme mít víc serverů. Pokud je jeden přetížen nebo neodpovídá včas, Nginx přepošle požadavek na jiný, volný server.

### Vykreslování obsahu a Jazyky
- **Staticky renderovaný web:** Server "hloupě" čte a posílá předem napsané .html soubory uložené na disku.
- **Dynamicky renderovaný web:** Generuje HTML "za pochodu" až ve chvíli, kdy o něj uživatel požádá, na základě dat z databáze (např. profil uživatele, košík e-shopu). K tomu slouží **jazyky interpretované na straně serveru**. Často se nepíšou od nuly, ale používáme **frameworky** (předpřipravené knihovny):
    - **Python:** Frameworky Django, Flask
    - **JavaScript (na straně serveru):** Frameworky Angular, Next.JS (běží nad Node.js)
    - **Ruby:** Framework Ruby on Rails
    - **PHP:** Celý jazyk PHP je už od základu navrhnut přímo pro použití na webu (pohání např. celý WordPress).

## Návod: Jak si sehnat webhosting a doménu (Publikování webu)
Když chceš mít skutečný web (např. `mojefirma.cz`), postupuješ takto:

**1. Výběr poskytovatele a ověření domény**
- Vybereš si poskytovatele (např. Wedos, Forpsi, Český hosting, Active24).
- Na jejich stránkách zadáš do vyhledávače název, který chceš (např. `muj-super-web.cz`), a systém ověří, zda už si ho nekoupil někdo jiný.

**2. Nákup balíčku (Doména + Hosting)**
- Vložíš doménu do košíku a přidáš k ní službu **Webhosting** (přidělí ti diskový prostor, např. 5 GB na SSD, a databázi).
- Obvykle se to platí na rok dopředu (doména .cz stojí cca 200 Kč/rok, základní hosting cca 400 Kč/rok).

**3. Propojení (DNS záznamy)**
- Pokud koupíš doménu i hosting u jedné firmy (což se doporučuje), propojí se vše automaticky. Prohlížečům pak chvíli trvá (až 24 hodin), než se nové DNS záznamy propíšou po celém světě.

**4. Nahrání souborů na server (přes FTP)**
- Poskytovatel ti pošle e-mail s přihlašovacími údaji k **FTP (File Transfer Protocol)**.
- Stáhneš si FTP klienta (např. bezplatný program **FileZilla**).
- V programu zadáš IP adresu serveru, jméno a heslo. Objeví se ti dvě okna: vlevo tvůj počítač, vpravo tvůj koupený webhosting.
- Jednoduše přetáhneš své soubory (`index.html`, `style.css`, `script.js`) zleva doprava do složky `/www`. 
- Od té chvíle je tvůj web online! *(Případně většina hostingů nabízí v administraci tlačítko "Instalovat WordPress" na jedno kliknutí, pokud nechceš kódovat od nuly).*

## Alternativní bezplatné nástroje pro vývojáře

### Live Server (Rozšíření do Visual Studio Code)
- Pokud otevíráš nakódovaný web jen z disku (`file:///`), musíš při každé úpravě kódu ručně obnovovat prohlížeč (F5).
- Nástroj **Live Server** ti na PC vytvoří lokální server (`localhost:5500`). Obsahuje funkci **Hot Reload** – jakmile v editoru uložíš kód (Ctrl+S), prohlížeč se na druhém monitoru automaticky sám obnoví.

### GitHub Pages (Hosting zdarma)
Pokud máš pouze statický web (jen HTML/CSS/JS, bez databáze a PHP), nemusíš si hosting kupovat:
1. Založíš si účet na GitHubu a vytvoříš veřejný repozitář.
2. Nahraješ tam své soubory (hlavní musí být `index.html`).
3. V nastavení repozitáře (Settings -> Pages) zapneš publikování z hlavní větve (`main`).
4. GitHub ti do pár minut vygeneruje trvalý odkaz (např. `tvojejmeno.github.io/tvuj-web`), který funguje zdarma napořád.


# HTML, JS, CSS

## 1. HTML (HyperText Markup Language)
HTML **není programovací jazyk**, ale **značkovací jazyk** definující strukturu a obsah webové stránky. 

- Vzhledově je sice podobný XML, ale **HTML nelze označit jako nadstavbu XML!** Zásadní rozdíl je v tom, že v HTML můžeme na rozdíl od striktního XML běžně používat neuzavřené (nepárové) tagy (např. `<br>`, `<img>`, `<input>`).

**Prázdný HTML dokument (základní šablona dle editoru Visual Studio Code):**
```html
<!DOCTYPE html>
<html lang="cs">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Název dokumentu</title>
</head>
<body>
    
</body>
</html>
```

**Základní a nejpoužívanější tagy:**
- `<h1>Hlavní nadpis</h1>` (Nejvyšší úroveň nadpisu, měl by být na stránce jen jednou).
- `<h2>Nadpis nižší úrovně</h2>`
- `<p>odstavec</p>`
- `<b>tučný text</b>`
- `<i>kurzíva</i>`
- `<br>` break (zalomení řádku, nepárový tag).
- `<a href="https://google.com" target="_blank">odkaz</a>` (Anchor tag pro hypertextové odkazy, atribut *href* určuje cíl, *target* otevírá v nové záložce).
- `<img src="obrazek.jpg" alt="Popis">` (Vložení obrázku, *src* je zdroj, *alt* je alternativní text).

**Sémantika a HTML5:**
Současným standardem je **HTML 5**. Ten klade obrovský důraz na tzv. **sémantické významy tagů**. To znamená, že tag nemá určovat jen vzhled, ale hlavně význam obsahu. Například se doporučuje nahradit čistě vzhledový `<b>` (bold) sémantickým tagem `<strong>` pro vyjádření důrazu při čtení (což ocení vyhledávače a správně to interpretují hlasové čtečky pro slepé).

## 2. CSS (Cascading Style Sheets)
Kaskádové styly **definují grafickou podobu stránky** (barvy, fonty, rozložení, velikosti, animace).

Základní ukázka formátování stránky:
```css
body {
    background-color: white; /* Barva pozadí */
    color: black;            /* Barva textu */
    padding: 10px;           /* Vnitřní odsazení */
}
```

### Jak prohlížeč pozná, co má obarvit? (Selektory, class a id)
Abychom v CSS mohli měnit vzhled konkrétních prvků z HTML, používáme **CSS Selektory** napojené na HTML atributy `class` a `id`.

- **Atribut `class` (Třída) = Skupinová jmenovka:**
  Používá se pro prvky, které mají vypadat stejně (např. všechna potvrzovací tlačítka nebo obrázky v galerii). Může jich být na stránce neomezeně. V HTML se píše jako `class="nazev"`, v CSS se na ni odkazujeme pomocí **tečky (`.`)**.
- **Atribut `id` (Identifikátor) = Unikátní rodné číslo:**
  Dané `id` se může na celé HTML stránce vyskytovat **pouze jednou jedinkrát**! Slouží pro unikátní sekce (např. hlavní navigační lišta, pole s výsledkem v kalkulačce). V HTML se píše jako `id="nazev"`, v CSS se na něj odkazujeme pomocí **mřížky (`#`)**.

*Příklad vazby mezi HTML a CSS:*
```html
<div id="hlavni-panel">
    <button class="tlacitko">Odeslat data</button>
    <button class="tlacitko">Zrušit</button>
</div>
```
```css
/* # vybere POUZE jeden unikátní prvek s id="hlavni-panel" */
#hlavni-panel { background-color: lightgray; } 

/* Tečka vybere VŠECHNY prvky, které mají class="tlacitko" */
.tlacitko { color: blue; } 
```

### Kaskádování a dědičnost (Inheritance)
Klíčovou vlastností CSS je, že **jednotlivé styly se vzájemně dědí a přepisují** (kaskáda). Pokud nastavíte vlastnost (např. barvu textu nebo font) rodičovskému prvku, jeho potomci (vnořené tagy) ji automaticky **zdědí**, dokud jim ji výslovně nepřepíšete. Vlastnosti ovlivňující rozměry (např. rámečky nebo odsazení) se nedědí.

*Příklad v CSS:*
```css
p {
    color: pink;
}
.cerveny-text {
    color: red; /* Specifičtější pravidlo přepíše zděděnou barvu */
}
```
*Příklad v HTML:*
```html
<p>
    This is to <em>illustrate</em> inheritance. A zde je <span class="cerveny-text">varování</span>.
</p>
```
*Vysvětlení:* Ačkoliv jsme obarvili narůžovo jen odstavec `<p>`, slovo "illustrate" obalené v tagu `<em>` (zvýraznění/kurzíva) bude také růžové, protože tuto vlastnost od odstavce **zdědilo**. Pouze tag `<span class="cerveny-text">` bude červený – jeho barvu jsme explicitně přepsali, což má v kaskádě přednost.

## 3. JavaScript (JS)
JavaScript je plnohodnotný **programovací jazyk interpretovaný na straně prohlížeče** (klienta).
- Dodává webu **interaktivitu a funkcionalitu** (výpočty, validace formulářů, oživení prvků).
- Využívá **úzké integrace s HTML a CSS** pomocí přístupu přes tzv. **DOM** (Document Object Model).
- **DOM** je rozhraní, které transformuje HTML strukturu dokumentu do objektů v JavaScriptu, aby se s nimi dalo programově manipulovat.

*Příklad manipulace s DOMem:*
```javascript
// Najde první element <em> na stránce a přepíše jeho textový obsah
document.querySelector("em").innerText = "Ahoj světe";
```

**Moderní vlastnosti JS:**
JavaScript nabízí plnou podporu **asynchronního programování** (např. stahování nových dat ze serveru na pozadí pomocí technologie AJAX / Fetch, aniž by se musela znovu načítat celá stránka) a **událostmi řízeného programování** (kód čeká a reaguje na události jako je kliknutí myší, stisk klávesy), které je pro web a webový prohlížeč absolutně žádoucí.

### 3. JavaScript Logika kalkulačky (Práce s DOM a Událostmi)
Zde přesně vidíme integraci s HTML přes DOM a událostmi řízené programování.

```javascript
// 1. UDÁLOST: Spustí se, až když je celé HTML vykreslené
document.addEventListener('DOMContentLoaded', () => {
  
  // 2. PRÁCE S DOMem: Transformace HTML prvků do JS objektů pomocí jejich ID
  const aEl = document.getElementById('vstup1');
  const bEl = document.getElementById('vstup2');
  const out = document.getElementById('vysledek');
  const opSpan = document.getElementById('opSymbol');

  const btnPlus = document.getElementById('tlPlus');
  const btnMinus = document.getElementById('tlMinus');
  const btnKrat = document.getElementById('tlKrat');
  const btnDeleno = document.getElementById('tlDeleno');

  // Funkce pro bezpečné zpracování textu z inputu na číslo
  function parseNum(s){
    if (typeof s !== 'string') return NaN;
    s = s.trim().replace(',', '.'); // Nahrazení české čárky za technickou tečku
    if (s === '') return NaN;
    return Number(s);
  }

  // Funkce, která mění DOM (zapisuje výsledek zpět do HTML)
  function show(text, isError = false){
    out.textContent = text;
    // Přidává/odebírá CSS třídu '.error' (pokud je isError pravda, text zčervená)
    out.classList.toggle('error', !!isError);
  }

  // Hlavní výpočetní logika
  function calc(op){
    opSpan.textContent = op; // Změní symbol mezi políčky
    const a = parseNum(aEl.value);
    const b = parseNum(bEl.value);

    // Ošetření chyb a dělení nulou
    if (isNaN(a) || isNaN(b)){
      show('Chybný vstup', true);
      return;
    }
    if (op === '÷' && b === 0){
      show('Nelze dělit nulou', true);
      return;
    }

    let r;
    // Provedení matematické operace na základě předaného parametru
    switch(op){
      case '+': r = a + b; break;
      case '−': r = a - b; break;
      case '×': r = a * b; break;
      case '÷': r = a / b; break;
      default: r = NaN;
    }
    show(format(r), false);
  }

  // 3. UDÁLOSTMI ŘÍZENÉ PROGRAMOVÁNÍ
  // Tlačítkům přiřadíme událost 'click'. Při kliknutí se zavolá funkce calc().
  btnPlus.addEventListener('click', () => calc('+'));
  btnMinus.addEventListener('click', () => calc('−'));
  btnKrat.addEventListener('click', () => calc('×'));
  btnDeleno.addEventListener('click', () => calc('÷'));
```
